# Hospital-Database-Creation-Data-Migration-System
This project replaces Excel-based hospital record systems with a secure, scalable relational database, improving data integrity, automation, and reporting capabilities.

Problem Statement
The legacy Excel system suffered from:
Duplicate and inconsistent records
No enforced relationships
Scheduling conflicts
No access control
Manual reporting

Solution Design
Unique identifiers for patients, doctors, departments, appointments
Foreign key constraints for referential integrity
Data validation rules to prevent invalid entries
Automated scheduling conflict prevention
Role-based access control for sensitive data
Department-wise revenue reporting

Tools Used
SQL
Relational database design
Data migration techniques

Business Value
Improved data security
Reduced operational errors
Enabled automated reporting and analytics
